    Re: Integrating christmas lights on own website?

    alright here we go:

    1. you need to find the required files. thats the crucial point about the whole thing, because you can only find them all by crawling into the code and this took me quite some time. but here you are:

    soundmanager flash files:
    /soundmanager2.swf
    /soundmanager2_flash9.swf

    ->put into an appropriate place on your webspace

    script files:
    /christmaslights.js
    /soundmanager2-nodebug-jsmin.js

    ->put into an appropriate place on your webspace

    for a "sound"-folder:
    /sound/glass0.mp3
    /sound/glass1.mp3
    /sound/glass2.mp3
    /sound/glass3.mp3
    /sound/glass4.mp3
    /sound/glass5.mp3

    ->create "sound"-directory at the same place as the "image" directory.

    for an "image"-folder:
    /image/bulbs-32x32-top.png
    /image/bulbs-50x50-fragments.png
    /image/bulbs-50x50-top.png
    /image/bulbs-64x64-top.png
    /image/bulbs-72x72-top.png
    /image/bulbs-96x96-top.png

    ->create "image"-directory at the same place as the "sound" directory.

    2. edit and insert this into the header of your site:
    HTML Code:
    <!-- CHRISTMAS LIGHTS -->
<script type="text/javascript" src="{path to soundmanager2-nodebug-jsmin.js"}></script>
<script type="text/javascript" src="http://ajax.googleapis.com/ajax/libs/yui/2.7.0/build/animation/animation-min.js"></script>
<script type="text/javascript" src="{path to christmaslights.js}"></script>
<script type="text/javascript">
var urlBase = '{path to directory with "sound" + "image" directory}/';
soundManager.url = '{path to directory with soundmanager flash files}/';
</script>
<!-- / CHRISTMAS LIGHTS -->
    if the variables soundmanager.url and/or urlBase are not set and left out, the default location for each will be:
    lights/
    (in relation to the scriptfiles)
    !Do not forget the trailing slash!

    for the YUI, instead of
    ajax.googleapis.com/ajax/libs/yui/2.7.0/build/animation/animation-min.js
    the default URL at the example from Scott Schiller is:
    yui.yahooapis.com/combo?2.6.0/build/yahoo-dom-event/yahoo-dom-event.js&2.6.0/build/animation/animation-min.js
    so first, we are using version "2.6.0", not "2.7.0" and haven't been running into problems with this.
    and second, we are loading the YUI from google instead from Yahoo.
    its your decision what version you wanna use and where to load it from. it doesn't make a difference.

    3. if wished, open christmaslights.js and edit the following line:
    Code:
      this.lightClass = (screenX>1280?'small':'pico'); // kind of light to show (32px to 96px square)
    (don't get confused, the comment isn't right)
    this means: screen resolutions with more than 1280 pixel in width will get lightbulbs in 'small' size, otherwise in 'pico' size.
    you can recognize the available sizes in pixels a few lines above this. you'll see:
    Code:
        pico: 32,
    tiny: 50,
    small: 64,
    medium: 72,
    large: 96
    currently we have set this to:
    Code:
      this.lightClass = (screenX>1280?'tiny':'pico');
    4. insert the following stylesheet-code (for example into your eventually already existing css-file) and edit the path at .xlsf-fragment in there:
    HTML Code:
    #lights {
 position:absolute;
 left:0px;
 top:0px;
 width:100%;
 height:100%;
 overflow:hidden;
}

.xlsf-light {
 position:absolute;
}

body.fast .xlsf-light {
 opacity:0.9;
}

.xlsf-fragment {
 position:absolute;
 background:transparent url({path to bulbs-50x50-fragments.png}) no-repeat 0px 0px;
 width:50px;
 height:50px;
}

.xlsf-fragment-box {
 position:absolute;
 left:0px;
 top:0px;
 width:50px;
 height:50px;
 *width:100%;
 *height:100%;
 display:none;
}

.xlsf-cover {
 position:fixed;
 left:0px;
 top:0px;
 width:100%;
 height:100%;
 background:#fff;
 opacity:1;
 z-index:999;
 display:none;
}
    5. done!
    if you have problems clicking links in an area under the lightbulbs, you have to edit
    height:100%;
    at #lights to a value that is appropriate, because at some browsers this div object overlays the normal site and so you cannot click anything.
    we have set this to 96px so it doesn't get in conflict with any site object.
    don't set it too tiny, otherwise the exploding fragments will disappear too early to look nice.


    i hope i haven't forgot anything, i didnt proof this guide. its just protocoll out from my memory.

    edit:
    insert the following code into the body at some place. best should be at the top:
    HTML Code:
    <!-- CHRISTMAS LIGHTS -->
<div id="lights">
  <!-- lights go here -->
</div>
<!-- / CHRISTMAS LIGHTS -->
